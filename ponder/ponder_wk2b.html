Ponder - Computational Thinking

  • Function

    Functions are a block of code that is designed to perform a particular task when something invokes or calls it.

    For example, when a user chooses a light or dark mode.

    We will learn more about functions later. Use the ponder activity for how to set up the function for this assignment.

  • Event Listeners

    The .addEventListener() method will run a function when the user does something. Or in other words will handle an event.

    For example, when a user chooses dark mode a function will be called that handles the logic needed to change the attributes to make it dark mode.

    We will learn more about event listeners later. Use the ponder activity for how to set up the event listener for this assignment.

  • Conditionals

    One conditional such as the 'if statement' specifies which block of code to execute based on different conditions.

    For example, depending on if the user choose dark or light mode different code will run.

    We will learn more about conditionals later. Use the ponder activity for how to set up the if clause for this assignment.

'Computational Thinking' is the way programmers think when they are solving problems with code. It helps the programmer plan what the program should do before they start writing the code. It's preparing step-by-step instructions that a computer can follow.

3 Steps of Computational Thinking:

  1. Break the problem into smaller parts
  2. Look for repeated or similar steps
  3. Make a list of steps

For example, say you want to write an application that lets the user choose their own theme or interface environment. They will choose from 3 different themes that have a different background and font style for the app.

This video demonstrates the idea that you have in your head:

Let's go through the Computational Thinking steps

  1. Break the problem into smaller parts
    • I need to give my user a way to choose the theme
    • Once they choose the theme, I need to change the theme according to what they chose.
  2. Look for repeated or similar steps
    • The process of choosing a theme will be repeatable if the user chooses another theme. They can change it repeatedly to different themes.
    • Two of the same properties will be changed: the background and the font for each theme.
  3. Make a list of steps
    • I'll have a drop-down menu to allow the user to choose a theme.
    • Listen for the drop-down menu option value to change.
    • When it changes retrieve the value they choose.
    • According to what theme they choose, change the background image and font style of my page to match that theme.

This last list of steps is also called pseudocode. Pseudocode uses plain language (or the language you speak and write in) along with programming logic. It won't run on a computer, so you don't have to have syntax perfect, but it helps you focus on logic and steps without worrying about code.

The same pseudocode, closer to the coding language

As you get to know your coding language better, your pseudocode might get a little closer to the programming language. For example:

  • Target all the elements I will need for this to run: the body element to change its background and font, and the drop down menu or select element.
  • Add an event listener to call a function to run code only when the user changes the theme.
  • In the function that runs when the theme changes:
    • Get the value from the drop-down so you know what theme they chose
    • If the value is ocean then change the body font style to Papyrus and the background image to ocean.jpg.
    • If the value is forest then change the body font style to Impact and the background image to forest.jpg.
    • If the value is desert then change the body font style to Big Caslon and the background image to dessert.jpg.
    • Otherwise no changes will be made to the background and the font will remain the same.

JavaScript:

          
let selectElem = document.querySelector('#theme-select');
let pageContent = document.querySelector('body');

selectElem.addEventListener('change', changeTheme);

function changeTheme() {
    let current = selectElem.value;
    if (current === 'ocean') {
        document.body.style.backgroundImage = "url('https://wddbyui.github.io/wdd131/images/ocean.jpg')";
        pageContent.style.fontFamily = "Papyrus, fantasy";
    } else if (current === 'forest') {
        document.body.style.backgroundImage = "url('https://wddbyui.github.io/wdd131/images/forest.jpg')";
        pageContent.style.fontFamily = "Impact, sans-serif";
    } else if (current === 'desert') {
        document.body.style.backgroundImage = "url('https://wddbyui.github.io/wdd131/images/desert.jpg')";
        pageContent.style.fontFamily = "'Big Caslon', serif";
    } else {
        // default
        document.body.style.backgroundImage = "none";
        pageContent.style.fontFamily = "Georgia, serif";
    }
}
