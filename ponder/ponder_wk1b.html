Ponder - JS Variables and Constants

  • Variables

    Variables are containers for storing data.

    Declare variables before you use them.

    The value assigned to a variable can change throughout code.

                            
// declare a variable called num and assign the value 6 to it
let num = 6;
// later in code you can reassign a new value to the variable
num = 10;

  • Constants

    Constants are containers for storing data.

    Declare Constants before you use them.

    The value assigned to a constant cannot change throughout code.

                            
// declare a const called password and assign the value 'R3d1ng!' to it
const password = 'R3d1ng!';

  • Scope

    A variable or constant has global scope if it's declared outside of any block of code.

    A varialbe or constant has block scope if it's declared inside of any block of code.

    Blocks of code can be functions, if statements, or loops. Anywhere a set of { } curly braces are used.

    If the variable or constant is declared within the curly braces it is block scope and can only be accessed inside that block. If the variable or constant is declared outside the block it can be accessed anywhere.

                            
let course = "CSE131"; //global scope
if (true) {
    let student = "John";
    console.log(course);  //works just fine, course is global
    console.log(student); //works just fine, it's being accessed with in the block
}
console.log(course); //works fine, course is global
console.log(student); //does not work, can't access a block variable outside the block

Script Tag

Use the script tag to link our JS file to our HTML.

Place the script tag either in the head portion of our HTML or right before the ending body tag.

If you place the script tag in the head portion of the HTML page you should use a defer attribute to allow the HTML page to load before the JS runs. This is critical when you reference any part of the HTML in the JS.

  <script src="week1.js" defer></script>
