Ponder - Computational Thinking

Computational Thinking is the way programmers think when they are solving problems with code. It helps the programmer plan what the program should do before they start writing the code. It's preparing step-by-step instructions that a computer can follow.

3 Steps of Computational Thinking:

  1. Break the problem into smaller parts
  2. Look for repeated or similar steps
  3. Make a list of steps

You want to write an application that lets the user choose their own theme or interface environment. They will choose from 3 different themes that have a different background and font style for the app.

This video demonstrated the idea that you have in your head:

Let's go through the Computationals Thinking steps

  1. Break the problem into smaller parts
    • I need to give my user a way to choose the theme
    • Once they choose the theme, I need to change the theme according to what they chose.
  2. Look for repeated or similar steps
    • The process of choosing a theme will be repeatable if the user chooses another theme. They can change it repeatedly to different themes.
    • Two of the same properties will be changed: the background and the font.
  3. Make a list of steps
    • I'll have a drop-down menu to allow the user to choose a theme.
    • Listen for the drop-down menu option value to change.
    • When it changes retrieve the value they choose.
    • According to what theme they choose, change the background image and font style of my page to match that theme.

This list of steps is also called pseudocode. Pseudocode uses plain language (or the language you speak and write in) along with programming logic. It won't run on a computer, so you don't have to have syntax perfect, but it helps you focus on logic and step without worrying about code.
